Hello,

# Change Request Policy

To avoid misunderstandings, I would define the project scope clearly in the agreement.

* Features listed in the approved scope are included in the quoted price.
* Any new functionality requested after development begins (for example, subscription plans, loyalty points, referral programs, mobile apps, or multi-city delivery) will be documented as a change request.
* Each change request will include an impact assessment covering additional cost, development time, and any effect on the project timeline.
* Work on change requests begins only after written approval from the client.

This approach keeps the project predictable for both parties.

This project is an excellent fit for **WordPress + WooCommerce**. It satisfies all of the current requirements, fits within the stated budget and timeline, and leaves room for future expansion.
